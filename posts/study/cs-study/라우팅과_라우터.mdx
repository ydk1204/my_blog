# 라우팅

라우팅은 데이터의 IP 패킷이 원본에서 대상으로 이동하는 방식을 말한다.
네트워크 라우팅은 하나 이상의 네트워크에서 경로를 선택하는 프로세스이다. 이는 전화 네트워크에서 대중 교통에 이르기까지 모든 유형의 네트워크에 적용될 수 있다.

인터넷과 같은 패킷 교환 네트워크에서 라우팅은 인터넷 프로토콜(IP) 패킷은 원본에서 대상으로 이동할 경로를 선택한다.
이러한 인터넷 라우팅 결정은 라우터라는 특수한 네트워크 하드웨어에 의해 이루어진다.

## 정적, 동적 라우팅

라우팅은 정적 라우팅과 동적 라우팅으로 나뉜다. 정적 라우팅 테이블은 변경되지 않으며, 네트워크 관리자가 수동으로 설정한다.
동적 라우팅 테이플은 자동으로 업데이트 되며, 다양한 라우팅 프로토콜을 사용하여 최단 경로와 가장 빠른 경로를 결정한다.

## 라우팅 프로토콜

주요 라우팅 프로토콜에는 IP, OSPF, RIP 등이 있다.

라우팅 프로토콜은 네트워크에서 데이터 패킷이 목적지까지 이동하는 경로를 결정하는 포로토콜이다.
이는 데이터가 어떤 경로를 따라 전달되어야 하는지를 결정하여 네트워크 효율성과 안정성을 유지한다. 주요 라우팅 프로토콜에 대해 좀 더 알아보자.

1. **IP** : IP는 인터넷 프로토콜의 약자로, 인터넷에서 데이터 패킷이 원본에서 대상으로 이동하는 경로를 결정하는 데 중요한 역할을 한다. IP 프로토콜은 데이터 패킷의 헤더 정보를 확인하여 어떤 경로를 통해 데이터를 전달해야 하는지를 결정한다. 이를 통해 데이터가 여러 네트워크를 거쳐 목적지로 이동할 수 있다. 주요 라우팅 프로토콜 중 하나인 OSPF는 IP를 기반으로 작동한다.

2. **Distance Vector 알고리즘** :
  - **RIP**(Routing Information Protocol): 표준 라우팅 프로토콜로 주로 사용된다. 라우터의 메모리 사용량이 적고, 30초마다 라우팅 정보를 업데이트한다. Hop count가 낮을 수록 좋은 경로로 간주되고, 소규모 네트워크에서 쉽게 구성할 수 있다.
  - IGRP(Interior Gateway Routing Protocol): 시스코에서 개발한 프로토콜로, 대규모 네트워크에서 적용 가능하다. 대역폭, 지연, 신뢰성, 부하, MTU를 고려하여 최적의 경로를 탐색한다.

3. **Link State 알고리즘** :
  - **OSPF**(Opne Shortest Path First): 표준 라우팅 프로토콜로 사용되며 SPF 또는 Dijkstra 알고리즘을 이용하여 최적 경로를 결정한다. Area 단위로 관리되며, EIGRP보다 대규모 네트워크를 안정적으로 운영한다.
  - EIGRP (Enhanced Interior Gateway Routing Protocol): 시스코에서 만든 독자적인 프로토콜이다. DUAL 알고리즘을 사용하여 최적 경로와 후속 경로를 결정한다. 하나의 네트워크 관리자가 인접한 네트워크 정보 이외에도 전체 경로를 아는 토폴로지 테이블을 가지고 있다.

# 라우터

라우터는 내부 라우팅 테이블을 참조해 네트워크 경로를 따라 패킷을 라우팅하는 방법을 결정한다. 라우팅 테이블에는 패킷이 라우터가 담당하는 모든 대상에 도달하기 위해 택해야 하는 경로가 기록된다. 이는 열차 시간표와 유사한 개념으로, 열차 승객이 어떤 열차를 타야 할지 결정하기 위해 참조하는 열차 시간표를 생각해보면 돼겠다. 라우팅 테이블도 이와 비슷하지만, 기차가 아닌 네트워크 경로를 위한 것이다.

라우터는 패킷의 헤더를 읽어서 가기 위한 목적지를 확인하고, 이를 기반으로 패킷을 라우팅할 위치를 결정한다.
라우터는 수백만 개의 패킷을 대상으로 초당 수백만 번의 작업을 수행하며, 패킷이 목적지로 이동할 때 각기 다른 라우터에 의해 여러 번 라우팅될 수 있다.
