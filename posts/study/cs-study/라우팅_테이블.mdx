# 라우팅 테이블이란

라우팅 테이블은 네트워크에서 목적지 주소를 해당 목적지까지 도달하기 위한 경로를 변환하는데 사용되는 데이터 구조이다. 이 테이블은 라우터나 다른 네트워크 장비가 목적지까지의 거리와 가는 방법을 결정하는데 필요한 정보를 포함하고 있다.

라우팅 테이블의 주요 구성요소는 다음과 같다.

---

# 라우팅 테이블 구성요소

1. 목적지 주소(Destination Address): 라우팅 테이블은 목적지 주소를 기반으로 경로를 설정한다. 이 주소는 목적지 호스트 또는 네트워크를 나타낸다.

2. 서브넷 마스크(Subnet Mask): 서브넷 마스크는 IP 주소의 네트워크 부분과 호스트 부분을 구분하는 역할을 한다. 라우팅 테이블은 서브넷 마스크를 사용하여 목적지 주소를 네트워크로 분류한다.

3. 게이트웨이(Gateway): 게이트웨이 목적지까지의 경로를 결정하는 중요한 요소이다 라우팅 테이블은 게이트웨이를 통해 목적지로 데이터를 전송한다.

4. 인터페이스(Interface): 라우팅 테이블은 목적지까지의 경로를 특정 인터페이스를 통해 전송한다. 각 인터페이스는 라우터 또는 네트워크 장비와 연결된 물리적 또는 논리적인 연결점이다.

5. 거리 (Metric): 라우팅 테이블은 목적지까지의 거리를 측정하는데 사용하는 메트릭 값을 포함한다. 이 값은 경로의 품질, 대역폭, 지역 시간 등을 고려해 결정된다.

라우팅 테이블은 네트워크 통신에 필수적이며, 목적지까지의 경로를 정확히 설정하는 것이 중요하다.

---

# 추가 정보

홉(hop): 홉은 컴퓨터 네트워크에서 출발지와 목적지 사이에 위치한 경로의 한 부분이다. 데이터 패킷은 브리지, 라우터, 게이트웨이를 거치면서 출발지에서 목적지로 경유한다. 각 패킷이 다음 네트워크 장비로 이동할 때마다 홉이 하나씩 발생한다. 이는 네트워크 통신에서 데이터가 여러 개의 중간 장치를 거쳐 목적지로 전달되는 과정을 비유적으로 표현한 것이다.

라우팅에서도 홉이 중요한 개념이다. 라우팅 프로토콜은 경로를 결정할 때 목적지까지 거치는 라우터의 수를 고려한다. 예를 들어, RIP(Routing Information Protocol)은 최대 15홉까지만 지원하며, 16홉을 넘어가는 경우 경로 비용을 무한대로 설정하여 해당 경로를 Unreachable로 처리한다. 이러한 홉 개념은 네트워크 통신에서 중요한 역할을 한다.
