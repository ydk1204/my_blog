# OSI 7계층이란?

OSI 7계층은 네트워크 통신을 위한 국제 표준 모델로, 데이터가 네트워크를 통해 전송되는 과정을 7개의 계층으로 나누어 설명한다. 각 계층은 다음과 같다.

## 1.Physical Layer

물리 계층(Physical Layer)

- 데이터를 전기 신호로 변환하여 전송하는 계층이다.
- 0과 1로 구분되는, 전기신호 ON/OFF를 떠올리면 된다.
- 전기신호를 주고 받는 것이 주 목적이며 데이터의 종류나 에러 유무는 확인하지 않는다.
- 데이터 전송 단위(PDU): 비트(Bit)
- 주요 프로토콜: X.21, RS-232 등
- 주요 장비: 허브(HUB), 리피터(Repeater), 네트워크 카드(NIC: Network Interface Card) 등

## 2.Data Link Layer

데이터 링크 계층(Data Link Layer)

- 물리 계층을 통해 안전하게 데이터를 전송하기 위한 프로토콜과 절차를 정의한다.
- 데이터 전송 시 데이터 전송 오류를 감지하는 기능을 제공하며 오류를 감지하면 재전송하는 방법으로 처리한다.
- MAC 주소를 가지고 통신한다.
- 데이터 전송 단위(PDU): 프레임(Frame)
- 주요 프로토콜: HDLC, X.25, Ethernet, TokenRing, DFFI, FrameRelay 등
- 주요 장비: 브리지(Bridge), L2 Switch 등

## 3.Network Layer

네트워크 계층(Network Layer)

- 데이터를 목적지까지 가장 효율적인 경로로 전송하는 기능을 담당한다.
- 주소(IP)를 정하고, 경로(Route)를 선택하고, 패킷을 전달하는 것이 가장 핵심인 계층이다.
- 데이터 전송 단위(PDU): 패킷(Packet)
- 주요 프로토콜: IP, ARP, ICMP, IGMP, RIP, RIP v2, OSPF, IGRP, EIGRP, BGP 등
- 주요 장비: 라우터(Router), L3 Switch 등

## 4.Transport Layer

전송 계층(Transport Layer)

- 데이터의 전송을 관리하며, 오류 검출 및 복구를 담당한다.
- 보통 TCP 프로토콜을 주로 사용한다.
- 데이터를 전송받은 경우, 전송계층에서 데이터를 합산하여 세션 게층으로 보내준다.
- 헤더에 송-수신 포트번호를 포함하여 전달하는 계층이다.
- 데이터 전송 단위(UDP):
  - TCP: Segment
  - UDP: Datagram
- 주요 프로토콜: TCP, UDP
- 주요 장비: L4 Switch

## 5.Session Layer

세션 계층(Session Layer)

- 통신 세션을 구성, 관리, 종료하는 기능을 수행한다.
- 통신 연결은 포트기반으로 구성하여 연결한다.
- OS가 세션 계층에 속한다.
- 데이터 전송 단위(PDU): 데이터(Data) 또는 메세지(Message)
- 주요 프로토콜: 없음
- 주요 장비: 없음

## 6.Presentation Layer

표현 계층(Presentation Layer)

- 데이터의 표현 형식을 정의하고, 암호화 및 변환을 담당한다.
- JPEG, TIFF, GIF, MPEG, 등 다양한 포멧을 구분한다.
- 데이터 전송 단위(PDU): 데이터(Data)
- 주요 프로토콜: 없음
- 주요 장비: 없음

## 7.Application Layer

응용 계층(Application Layer)

- 최종 사용자에게 서비스를 제공하는 계층으로, 네트워크 소프트웨어 UI와 같은 응용 프로그램을 포함한다, 따라서 사용자에게 보이는 유일한 계층이다.
- 메일 전송, 인터넷 접속 등의 작업을 수행할 수 있다.
- 데이터 전송 단위(PDU): 데이터(Data)
- 주요 프로토콜: TELNET, FTP, SMTP, HTTP 등
- 주요 장비: 없음
<br/>
<br/>
<br/>

이 계층들은 네트워크 통신의 복잡성을 단순화하고, 다양한 통신 기술과 프로토콜 간의 호환성을 보장하기 위해 사용된다.
