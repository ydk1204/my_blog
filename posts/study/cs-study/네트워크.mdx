# 네트워크

네트워크란 두 대 이상의 컴퓨터나 기타 장치들이 통신 매체를 통해 서로 데이터를
주고 받을 수 있도록 **연결**된 것이다. 네트워크는 컴퓨터나 장치들이 서로 정보나 자원을
공유하고 협업하고 상호작용할 수 있게 해준다.

네트워크는 크기나 범위, 구성, 프로토콜, 목적 등에 따라 다양한 종류로 분류될 수 있다.
예를 들어, LAN, WAN, MAN, PAN, VPN, WLAN, SAM 등이 있다.

---

# 노드와 링크

## 노드(node)

네트워크에서 노드(node)란 네트워크 상의 다른 노드로 데이터를 전송하거나
수신할 수 있는 모든 **장치**를 말한다.

예를 들어, 컴퓨터, 프린터, 라우터, 스위치 등이 노드가 될 수 있다.

## 링크(link)

노드들을 서로 연결하는 통신 **채널**을 말한다.

링크는 유선이나 무선으로 구분할 수 있다. 유선 링크는 동축 케이블이나 광섬유 등을
이용하고, 무선 링크는 라디오 주파수나 마이크로파 등을 이용한다.

# 그 외의 용어들

## 1. MAC 주소 :

- 네트워크 장치의 **물리적**인 식별자로, 48bit의 16진수로 표현되는 주소다.
- MAC 주소는 네트워크 카드 제조사에 의해 할당되고, **고유**하게 보장된다.
- MAC 주소는 **데이터 링크 계층**에서 **프레임**을 전달할 때 사용된다.

## 2. IP 주소 :

- 네트워크 장치의 **논리적**인 식별자로, 32bit의 10진수로 표현되는 주소다.
- IP 주소는 네트워크 관리자에 의해 할당되고, 네트워크와 호스트를 구분할 수 있다.
- IP 주소는 **네트워크 계층**에서 **패킷**을 전달할 때 사용된다.

## 3. 라우터 :

- **네트워크 간에** 패킷을 전달하는 장치
- **IP 주소**를 기반으로 최적의 경로를 결정하고, 라우팅 테이블을 관리한다.
- 라우터는 **네트워크 계층**에서 동작하고, 네트워크의 연결성과 확장성을 높여준다.

## 4. 스위치 :

- 네트워크 **내에** 패킷을 전달하는 장치
- **MAC 주소**를 기반으로 스위칭 테이블을 관리하고, **충돌 도메인**을 줄여준다.
- 스위치는 **데이터 링크 계층**에서 동작하고, 네트워크의 성능과 효율을 높여준다.

## 5. 허브 :

- 네트워크 내에 신호를 증폭하고 분배하는 장치
- 물리적인 **별형 구조**를 구성할 수 있다.
- 허브는 물리 계층에서 동작하고, **모든 포트**에 동일한 신호를 전달한다.
- 허브는 스위치보다 간단하고 저렴하지만 **충돌 도메인이 커서** 성능이 낮다.

## 6. 리피터 :

- 네트워크에서 신호를 수신하고 증폭하며 다음 구간으로 재전송하는 장치
- 리피터는 케이블의 **거리 제한을 극복**하고 네트워크 반경을 **연장**하는 역할을 한다.
- 리피터는 물리 계층에서 동작하고, 데이터의 내용이나 목적지를 구분하지 않고 단순히
**신호를 재생**하는 기능만 한다.

---

# 장치들의 차이점

## 스위치와 라우터

스위치와 라우터는 네트워크 장비로서 서로 다른 역할과 기능을 수행한다.

### 차이점
- 스위치는 같은 네트워크 내의 여러 장치를 연결하고, **MAC 주소**를 기반으로 데이터를 전달한다.
- 라우터는 서로 다른 네트워크 간에 데이터를 전달하고, **IP 주소**를 기반으로 최적의 경로를 결정해준다.

<br/>
스위치와 라우터를 구분해서 사용하는 상황은 다음과 같다.

### 사용하는 경우

1. 스위치
- 네트워크의 규모가 작고, 인터넷 연결이 필요하지 않은 경우에는 스위치만 사용할 수 있다.
- 스위치는 로컬 네트워크에서 장치들을 연결하고, 데이터를 공유하고, 통신하게 해준다.

2. 라우터
- 네트워크의 규모가 크고, 인터넷 연결이 필요한 경우에는 스위치와 라우터를 함께 사용한다.
- 스위치는 로컬 네트워크에서 장치들을 연결하고, 라우터는 로컬 네트워크와 인터넷이나 다른 네트워크를 연결한다.

<br/>

## 허브와 리피터

허브나 리피터는 둘 다 네트워크에서 신호를 증폭하고 분배하는 장치이지만,
다소 차이가 있다.

### 차이점

1. 포트
- 허브는 리피터보다 포트가 더 많아서 **여러 대의 컴퓨터나 장치를 연결**할 수 있다.
- 리피터는 보통 두 개의 포트만 가지고 있어서 **일대일**로만 연결할 수 있다.

2. 추가 기능
- 허브는 **패킷 모니터링이나 문제 고립**과 같은 추가적인 기능을 제공할 수 있다.
- 리피터는 단순히 신호를 재생 하는 기능만 수행한다.

3. 가격
- 허브는 리피터보다 가격이 비싸다.
- **리피터**는 가장 저렴한 네트워크 장비 중 하나이다.

<br/>
허브와 리피터를 구분해서 사용하는 상황은 다음과 같다.

### 사용하는 경우

1. 더 멀리
- 네트워크의 거리를 연장하고 싶을 때는 **리피터**를 사용할 수 있다.
- 리피터는 케이블의 **거리 제한을 극복**하고 네트워크 반경을 **연장**하는 역할을 한다.

2. 여러개
- 네트워크에 여러 대의 컴퓨터나 장치를 연결하고 싶을 때는 **허브**를 사용할 수 있다.
- 허브는 **멀티포트**를 지원하고, 물리적인 **별형 구조**를 구성할 수 있다.

다만, 요즘은 허브나 리피터보다 스위치나 라우터와 같은 고급 네트워크 장비들이
많이 사용되고 있다. 스위치나 라우터는 허브나 리피터보다 더 빠르고 효율적이고 안전
하게 네트워크를 구성할 수 있다.

# 번외(중간의 용어들)

### 충돌 도메인

위에서 나온 충돌 도메인은 무엇일까?

충돌 도메인이란 네트워크에서 프레임이 충돌할 수 있는 장치들의 범위를 말한다.
예를 들어, 허브나 리피터로 연결된 이더넷 세그먼트는 하나의 충돌 도메인이다.
충돌 도메인은 스위치나 브리지로 분할할 수 있다고 한다.

충돌 도메인이 작을 수록 네트워크의 성능이 좋아진다.

<br/>

### 프레임

프레임은 차후 또 나오는 단어일텐데, 간단하게 알아보자.

네트워크에서 데이터를 전송할 때 사용되는 **기본 단위**라고 볼 수 있다.

헤더와 트레일러를 포함한 일련의 비트들을 말한다. 프레임은 데이터의 출처와
목적지, 데이터의 형식, 데이터의 무결성 등을 나타내는 정보를 가지고 있다.

프레임은 데이터 링크 계층에서 생성되고, 네트워크 장비들을 통해 전달되고,
수신 측에서 해석되어 상위 계층으로 전될되는 과정을 거친다.

<br/>

### 패킷

패킷은 위의 프레임과 비슷하다.

패킷은 네트워크에서 데이터를 전송할때 사용되는 작은 정보 단위이다.

패킷은 헤더와 페이로드로 구성되어 있다. 헤더에는 패킷의 출처와 목적지, 크기,
순서 등에 대한 정보가 있고, 페이로드에는 실제 데이터가 있다.

패킷은 인터넷과 같은 패킷 스위칭 네트워크에서 서로 다른 경로를 통해 전달되고,
수신 측에서 재조립되어 원본 메시지를 복원한다.

<br/>

#### 패킷과 프레임의 차이점

1. 계층

- 패킷은 네트워크 계층에서 사용되는 데이터 단위이다, IP 주소와 같은 논리적인 주소를 사용한다.
- 프레임은 데이터 링크 계층에서 사용된다, MAC 주소와 같은 물리적인 주소를 사용한다.

2. 내외부

- 패킷은 네트워크 간에 데이터를 전달할 때 사용한다.
- 프레임은 네트워크 내에 데이터를 전달할 때 사용한다.
