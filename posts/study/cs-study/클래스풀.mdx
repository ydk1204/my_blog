IP 주소를 보는 것은 이제 많이 흔해서 익숙하다. 근데 IP 주소는 어떤 방식으로 정해진 것 일까?

IP 주소라는 명칭에 맞게 현실의 동네 주소처럼 정해진 체계가 있을 것 이다. 그럼 이 주소 체계는 어떻게 정의되어 있을까?<br/>
그 IP 주소 체계에 대한 주제가 클래스풀(Classful IP Addressing)이다.

---

# 클래스

클래스풀(Classful IP Addressing)은 인터넷 프로토콜(IP) 주소 체계를 클래스별로 구분하여 정의하는 방식이다.
이 방식은 IP 주소를 **네트워크 주소**와 **호스트 주소**로 나누고, 이를 클래스 A, B, C, D, E로 구분한다.

> 네트워크 주소가 동일하면 로컬 네트워크<br/>
호스트를 구분하기 위한 주소는 호스트주소 <br/>
네트워크 호스트(network host)는 컴퓨터 네트워크에 연결된 컴퓨터나 기타 장치

참고로 IP 주소를 보면 8비트로 나뉘어 지는데, 이 8비트 블록을 **"옥텟"**이라 부른다.<br/>
IP 주소는 32비트로 구성되고, 이를 4개의 옥텟으로 나누어 표현한다고 볼 수 있다.<br/>
각 옥텟은 0부터 255까지의 값을 가질 수 있다. 예를 들어, 127.115.255.255 에서 첫 번째 옥텟은 127,
두 번재 옥텟은 115, 세 번재와 네 번째 옥텟은 255 이다. 이렇게 옥텟을 조합해 IP 주소를 만든다.

## 1. 클래스 A

첫 번째 옥텟의 가장 왼쪽 비트가 0으로 시작하는 주소로, 범위는 0.0.0.0 ~ 127.255.255.255이다.

여기서 잠깐 왼쪽 비트가 0부터 시작하는 주소라고 했는데 왜 127까지 일까?
1. 위의 주소는 우리가 보기 편한 10진수로 표기한 방식이다.
2. 왼쪽 "비트"가 0부터 시작이라고 했음으로 2진수로 생각해야 한다.

따라서 위의 주소 범위를 10진수가 아닌, 2진수로 다시 표기한다면 다음과 같이 된다.<br/>
0.0.0.0 =         00000000.00000000.00000000.00000000<br/>
    ~<br/>
127.255.255.255 = 01111111.11111111.11111111.11111111

> 추가로 0.0.0.0 도 클래스 A의 주소 범위라 적었지만 0.0.0.0은 특수주소(알 수 없는 대상에 달아 놓는 임시 주소)라 포함하지 않는다.<br/>
> 비슷한 결로 127.255.255.255도 원래는 포함하지만 127.xxxx.xxxx.xxxx은 루프백 주소이기 때문에 포함하지 않는다.(웹 개발할 때 localhost 주소를 많이 봤을 것 이다.)

범위는 알았으니 위에 말한대로 네트워크 주소와 호스트 주소를 나눠보자.

### 네트워크/호스트(A)

가장 보기 좋은 주소인 0.0.0.0을 예시로 보자.

00000000.00000000.00000000.00000000을<br/>
0XXXXXXX.ZZZZZZZZ.ZZZZZZZZ.ZZZZZZZZ 으로 변환해서 볼 때,<br/>
X가 네트워크 주소 범위이고, Z가 호스트 범위이다.

즉, 네트워크 주소(X)는 맨 왼쪽의 옥텟에서 1개의 0을 제외한 7개의 비트(Bit)를 가지고,
호스트(Z)는 그외 나머지 24비트를 가진다.(=8x3)
- 2^24-2 = 한 네트워크당 16,777,214 호스트 ID(약 1600만개) / 0

주소 2개를 빼는 이유는 맨 앞자리는 네트워크 주소로 남겨두고, 마지막 주소는 브로드캐스팅 주소로 남겨 놓기 때문이다.
따라서 -2를 한다. 이는 A, B, C 클래스 모두 해당한다.

클래스 A는 전체적인 설명이 존재하여 길었지만, B부터는 똑같은 내용이니 짧게 서술한다.

---

## 2. 클래스 B

첫 번째 옥텟의 첫 번째 비트가 1로 시작하는 주소로, 범위는 128.0.0.0 ~ 191.255.255.255 이다.

위의 주소 범위를 10진수가 아닌, 2진수로 다시 표기한다면 다음과 같이 된다.<br/>
128.0.0.0 =       10000000.00000000.00000000.00000000<br/>
    ~<br/>
191.255.255.255 = 10111111.11111111.11111111.11111111

### 네트워크/호스트(B)

10000000.00000000.00000000.00000000을<br/>
10XXXXXX.XXXXXXXX.ZZZZZZZZ.ZZZZZZZZ 으로 변환해서 볼 때,<br/>
X가 네트워크 주소 범위이고, Z가 호스트 범위이다.

즉, 네트워크 주소(X)는 1옥텟의 1과 0을 제외한 6비트와, 2옥텟의 모든 비트(8비트)를 더한 14비트를 가지고,
호스트(Z)는 그외 나머지 16비트를 가진다.(=8x2)
- 2^16-2 = 한 네트워크당 65534 호스트 ID(약 6만 5천개) / 10

---

## 3. 클래스 C

첫 번째 옥텟의 첫 번째와 두 번째 비트가 1, 세번째 비트가 0으로 시작하는 주소로, 범위는 192.0.0.0 ~ 223.255.255.255 이다.

위의 주소 범위를 10진수가 아닌, 2진수로 다시 표기한다면 다음과 같이 된다.<br/>
192.0.0.0 =       11000000.00000000.00000000.00000000<br/>
    ~<br/>
223.255.255.255 = 11011111.11111111.11111111.11111111

### 네트워크/호스트(C)

11000000.00000000.00000000.00000000을<br/>
110XXXXX.XXXXXXXX.XXXXXXXX.ZZZZZZZZ 으로 변환해서 볼 때,<br/>
X가 네트워크 주소 범위이고, Z가 호스트 범위이다.

네트워크(X)는 21비트를 가지고,
호스트(Z)는 그외 나머지 8비트를 가진다.(=8x1)
- 2^8-2 = 한 네트워크당 254 호스트 ID / 110

---

## 4. 클래스 D

첫 번째 옥텟의 첫 번째, 두 번째, 세 번째 비트가 1, 네 번째 비트가 0으로 시작하는 주소로, 범위는 224.0.0.0 ~ 239.255.255.255 이다.

위의 주소 범위를 10진수가 아닌, 2진수로 다시 표기한다면 다음과 같이 된다.<br/>
224.0.0.0 =       11100000.00000000.00000000.00000000<br/>
    ~<br/>
239.255.255.255 = 11101111.11111111.11111111.11111111

### 특수 주소(D)

클래스 D와 E는 네트워크 주소와 호스트 주소를 나누지 않는다. 이 두 클래스는 특별한 용도로 사용되며, 일반적인 호스트 주소 할당에는 적합하지 않다.

클래스 D 주소는 **멀티캐스트** 그룹을 지원하기 위해 사용된다. 멀티캐스트는 하나의 송신자가 여러 수신자에게 데이터를 전송하는 방식으로,
예를 들어 동일한 데이터를 여러 컴퓨터에 동시에 전송할 때 사용된다.

---

## 5. 클래스 E

범위 240.0.0.0 ~ 255.255.255.255 를 가진다.

위의 주소 범위를 10진수가 아닌, 2진수로 다시 표기한다면 다음과 같이 된다.<br/>
224.0.0.0 =       11110000.00000000.00000000.00000000<br/>
    ~<br/>
255.255.255.255 = 11111111.11111111.11111111.11111111

### 특수 주소(E)

클래스 E 주소는 **실험, 연구, 테스트** 등의 목적으로 예약되어 있다. 실제 네트워크에서는 거의 사용되지 않는다.

---

# 클래스풀의 문제점

**확정성**과 **낭비**의 문제가 있다.

예를 들어 네트워크의 크기가 작은 경우 큰 네트워크를 필요로 하는 조직은 여러개를 확보해야 하고,
작은 네트워크가 필요한 조직의 경우 너무 많은 IP를 가져갈 수 있기 때문에 IP가 낭비되는 문제점이 있다.
