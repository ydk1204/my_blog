## SEO를 한번 알아보자

### SEO가 머임??

SEO - Search Engine Optimization(검색 엔진 최적화)  
웹사이트가 검색 결과에 더 잘 보이도록 최적화하는 과정이며, 검색 랭크 개선이라고도 한다.

### SEO 그 친구를 신경 써야되는 이유

검색 엔진은 웹을 크롤링 하면서 찾은 콘텐츠의 색인을 생성하는데,  
검색 결과에 보이는 것이 바로 그 콘텐츠 색인이다.

우리는 이 검색 엔진이 크롤링을 할 때 우리가 작성한 글이 검색 결과에서 높은 위치에 노출되게
하기 위해서 SEO를 신경 써야한다.  
그 이유는 당연히 공들여서 쓴글을 보다 많은 사람들이 보길 바라는 마음이 보통이기 때문이다.

간단하게 SEO가 무엇인지 왜 필요한지 알았으니 살짝 더 자세히 들여다보자.

## 용어 설명

우리는 위에서 봤던 내용과 이후 나오게 될 몇 가지 단어의 뜻을 먼저 알고 가기로 한다.

아래의 내용은 [Google Webmasters 학습 문서][구글 디벨롭 URL]의 내용을 정리했다.

### 색인

- Google에서는 알고 있는 모든 웹페이지를 색인에 저장한다.
- 페이지의 색인 항목은 해당 페이지의 콘텐츠 및 위치(URL)를 명시한다.
- Google에서 페이지를 가져와서 읽고 색인에 추가하면 색인 생성 끝

난 이것만 봐서는 잘 모르겠어서 좀 더 찾아봤다.

- 코딩할 때 자주 봤었던 index가 색인이라고 한다. 처음 알았다.
- 색인은 데이터를 기록할 대 데이터의 이름이나 크기 등의 데이터를 보여주는 속성과 장소 등을
  보기 쉽게 표시하는 것
- 즉, 내가 어떤 데이터를 만들었다면 그 데이터에 대한 참조가 될 수 있는 것, 그것이 색인이다.
- 서점이나 도서관에 가면 책이 많이 있는데, 이 책의 내용의 빠르게 확인하기 위해 간단하게 목차를 훑어 보는 느낌인 것 같다.

색인을 위처럼 책의 목차로 생각해본다면 다음과 같이 생각할 수도 있을 것 같다.  
서점 = 구글, 책 구매자 = 검색자, 글쓴이 = 글쓴이

1. 서점에 베스트 셀러로 팔리는 책이 있다.
2. 서점에서 그 책의 내용과 비슷한 책을 근처에 배치하면 다른 책도 잘 팔릴 수 있으니
   그 근처에 둘 다른 책을 찾는다.
3. 빠르게 다른 책들의 목차를 살피며 비슷한 종류의 책을 찾아서 배치해둔다.
4. 책 구매자가 와서 베스트셀러의 책을 보고 그 근처에 있던 비슷한 종류의 책들도 한번씩 둘러본다.

실제 방식은 어떤지 모르겠지만 위와 같이 생각한다면 색인의 중요성이 왜 높은지 대충 생각해볼 수 있을 것 같다.

서점(구글)은 더 다양한 콘텐츠를 제공함으로써 매출을 증가시키고,  
책 구매자(검색자)는 빠르게 다양한 콘텐츠를 살펴 볼 수 있으니 편하고,
글쓴이는 아무튼 이득이다.

결론 = 색인 최고네

### 크롤링

신규 또는 업데이트된 웹 페이지를 찾는 프로세스이다.
이 친구는 자동화된 프로그램으로 게임으로 비유하자면  
자동 사냥 매크로처럼 자동으로 새로 만들어진 웹 페이지를 찾아주는 프로그램이다.

크롤링은 Google링크를 따라가거나, 사이트맵을 읽는 등 여러 방법으로 URL을 찾아낸다고 한다.

그래서 찾은 다음에 무엇을 하냐, 위에서 본 색인을 만든다.

### 크롤러

위에서 본 크롤링이 웹 페이지를 찾는 프로그램이면, 이 친구는ㄴ 색인을 생성하는 프로그램이다.

즉, 웹에서 페이지를 크롤링(찾음, 가져옴)한 다음 크롤러가 색인을 생성한다.

### Googlebot

Google 크롤러의 일반적인 이름이라고 한다.

## 검색 엔진 최적화의 분류

SEO를 찾아보던 중 [위키백과][위키백과]도 살펴봤다.  
대부분의 내용이 타 사이트의 내용과 비슷했지만 달랐던 점도 존재했다,
"검색 엔진 최적화의 분류" 바로 이 부분인데,

검색 엔진 최적화의 분류에는 크게 두가지로 분류된다고 한다.

1. 온 페이지 SEO(On-Page SEO)
2. 오프 페이지 SEO(Off-Page SEO)

이 분류 방법은 웹 사이트 내부에서 검색 엔진 최적화를 하는지 외부에서 하는지에 따라 나누는 것으로 보인다.

1.온 페이지 SEO의 경우 내부에서 최적화를 하는 것으로

- H태그
- 메타태그
- 이미지 최적화
- 콘텐츠 최적화
- 글자 수
- 내부 링크
  등의 요소들이 포함

  2.오프 페이지 SEO의 경우 외부에서의 최적화를 이루는 말로

- 관련성
- 신뢰성 및 권한
- 사이트로 향하는 백링크
  등이 주된 작업이라고 한다.

여기서 *신뢰성*은 무엇을 말하는지 궁금했는데, 위키백과의 SEO 문서 본문에
다음과 같은 글이 적혀있었다.

> SEO의 기본적인 작업 방식은 검색어를 웹 페이지에 적절하게 배치하고
> 다른 웹 페이지에서 **링크**가 많이 연결되도록 하는 것이다.  
> 구글 등장 이후 검색 엔진들이 콘텐츠의 *신뢰도*를 파악하는 기초적인 지표로 다른 웹사이트에
> 얼마나 **인용**되었나를 사용하기 때문에 타 사이트에 인용되는 횟수를 늘리는 방향으로 최적화한다.

이 문장에서 *신뢰도*이라는 단어가 나오기 때문에 위 문장을 보고 오프 페이지 SEO에서 말한 *신뢰성*을 생각해 보자면,
사람들이 많이 보고 참고한 글이 곧 *신뢰도*가 높다라고 생각할 수 있을 것 같다.

## 그렇다면 SEO를 어떻게 하면 좋을까?

간단하게 SEO가 무엇인지, 어떤 방식으로 구글이 여러 콘텐츠들을 살펴보는지 알았다.

하지만 가장 중요한 것은 그래서 대체 어떻게 해야 검색엔진을 최적화 할 수 있는가! 이것이다.
사실 위의 내용은 다 상관없고 이게 가장 궁금했다.

보통 10가지의 방법을 소개하는 것으로 보인다. 근데 그 방식이 조금씩 달라
여기서는 10가지를 다 채우지는 않고 정말 필요해 보이거나 공통으로 보이는 몇가지를 적는다.

### 1.문법에 맞는 HTML 작성하기

- 몇번 들어봤을 수 있는 시멘틱 태그를 말한다.
- &#60;title&#62; 태그에는 사이트 제목,
  &#60;strong&#62;과 &#60;em&#62;태그를 이용해 강조 표시 등  
   적절한 HTML 태그를 이용해 웹 페이지를 구성하는 것은 검색엔진이 살펴보기 쉽게 만들어 준다.

### 2.HTTPS 보안 프로토콜 사용

- 구글은 HTTPS 보안 프로토콜을 사랑한다고 하니 참고하자

### 3.robots.txt 사용

- robots.txt는 사이트맵의 위치와 접근 가능한 파일, 그렇지 않은 파일 등의 정보를 알려준다.
- 이 robots.txt 파일은 직접 스크립트를 만들어서 사용할 수도 있지만, "next-sitemap"이라는 패키지를 다운받아서
  편하게 사용할 수 있다.

### 4.사이트맵

- 사이트맵은 웹 사이트 내의 모든 페이지의 URL을 하나의 파일에 정리한 것이라고 생각하면 된다.
- 위에서 살펴본 크롤러가 색인할 리스트라고 보면 된다.

  - 쉽게 생각하자면 크롤러가 보게 될 지도나 체크리스트라고 생각하면 될 것 같다.

### 5.메타 태그와 디스크립션 활용하기

- HTML 문서를 만들었다면 한번씩 봤던 &#60;head&#62;태그 내에 있는 &#60;meta&#62;태그를 의미한다.

```
<meta name="keywords" content="키워드1">
<meta name="description" content="본문 요약">
```

- 위의 코드 같이 적절히 사용한 &#60;meta&#62;태그와 description은 검색엔진의 검색 결과에 반영되어 표기되기 때문에 포함하는게 좋다고 한다.
- 추가로 모든 웹 페이지에 똑같은 description을 적용하는 것은 효과적이지 않으니 각 페이지 별로 다르게, 본문의 내용을 요약한 정보를 넣는 것을 추천한다.

### 6.캐노니컬 태그

- 캐노니컬 태그는 &#60;meta&#62;태그와 같이 &#60;head&#62; 태그 안에 삽입하는 태그이다.
- 특정 웹페이지의 대표 url 주소를 검색엔진에게 알려주는 역할을 한다.

### 7.이미지 alt 속성 명시

- alt란, alternative의 약자로 이미지가 로딩되지 못했을 때 대힌 표시되는 문자이다.
- alt 속성에 값을 할당해야 되는 이유는 다음과 같다.

  - 시각장애인의 경우 스크린리더를 사용하는데 이 스크린리더의 경우 이미지 대신 alt 속성에 부여된 값을
    읽어 들인다. 이를 통해 해당 파트에 어떤 이미지가 있는지 파악할 수 있어 이미지 사용시 alt 속성을 사용하도록 하자.
  - 검색엔진도 alt 속성에 부여된 값을 통해 인덱싱(색인) 작업을 하기 때문에 alt 속성은 반드시 필요하다.

### 8.페이지 로딩 속도 개선

- 페이지 로딩 속도는 검색엔진최적화를 결정하는 요소 중 매우 중요한 부분을 차지한다고 한다.
- 구글은 이 로딩 속도가 짧은 웹사이트에 더 높은 SEO 점수를 준다.
- 크롬브라우저의 경우 Lighthouse라는 확장 프로그램이 있는데 이를 사용하면 현재 페이지의 성능, 접근성, SEO 등을 간단하게 진단해준다.
  ![LightHouse 실행 이미지](/postImg/html/lighthouse.png)  
   이 글을 적고 있는 현재 시점에 블로그 메인에서 Lighthouse를 사용했을 때의 화면이다.  
   성능 부분이 좀 많이 아쉽긴 하지만... 아무튼 위 사진과 같이 간략한 점수 외에도 왜 저런 점수가 나왔는지 등의 자세한 내용이 나오니
  많은 도움이 될 것이다.

### 9.모바일 친화성

- 2016년 말부터 구글에서는 사이트 콘텐츠의 모바일 버전을 주로 사용하여 순위를 지정하고 구조화된 데이터를 파싱하며 스니펫을 생성하는 실험을 시작했다.
- 위 내용은 [Google Webmasters 학습 문서][구글 디벨롭 URL]에 적혀 있는 내용 중 일부이다.
- 즉 해당 내용을 살펴보면 구글 검색엔진최적화는 모바일 중심으로 색인을 생성한다고 볼 수 있겠다.
- 그렇기 떄문에 처음부터 모바일에서 보기 좋게 만들던가, 반응형으로 만들면 좋겠다.
- 어쩐지 위에서 사용했던 Lighthouse의 결과 화면도 모바일 화면으로 보였던 이유가 여기있는 것 같다.

어쩌다 보니 9가지를 적게 되었는데 10개가 아니라 뭔가 스스로도 찝찝한 느낌이 든다.
하지만 어쩌겠는가 더 보고 싶다면 검색해보는 것을 추천한다. 검색하면 많이 나온다.

이상으로 SEO에 대해 알아봤다.  
몇가지는 대략 알고 있던 점이 있었는데 캐노니컬 태그의 경우 이번 글을 작성하면서 처음 알게 됐다.

주로 검색을 구글에서 진행하다 보니 구글을 기준으로 글을 작성하게 됐는데,  
네이버의 경우 몇가지 좀 더 특별한 방식이 있는 것 같기도 하고 구글도 위에 적은 것보다 훨씬 많은 추천 방식이 있다.
구글의 경우 위에 적은 구글 가이드 페이지에 아주아주 상세한 설명이 적혀있으니 시간이 넉넉하다면 한번 보도록 하자,
물론 나는 너무 길어서 다 안 읽었다.

마지막으로 이 글을 작성하면서 참고한 글들의 링크를 남기며 마무리하도록 한다.
